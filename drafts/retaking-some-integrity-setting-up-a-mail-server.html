Retaking some integrity: Setting up a mail server

Posted by Simon Larsén in Linux   

As of late, I've been getting more and more concerned with privacy and integrity, something of which there is precious little of these days. As a first step, I've decided to try my hand at running my own mail server, complete with calendar functionality and webmail. It's actually not that hard to get started with, there are just a few hoops to jump through, and some things to keep in mind. Also, the real trouble with a mail server is not setting it up, but keeping it running. Only time will tell if I can keep mine running.

  • Setup CentOS 7 image on Linode
    • Create a user
    • Disable root login
    • Add trusted key
    • Disable login with password auth
  • Install iRedMail
    • Download latest version
    • Run install script
    • Pick whatever you want (I picked SOGo netdata iRedAdmin Fail2ban)
    • For the mail domain, just pick whatever subdomain you own. You'll automatically get a postmaster@domain mail address generated, where domain is the domain you gicked.
  • Generate a proper TLS cert
    • Install certbot
    • Generate a new certificate with certbot certonly -d DOMAIN
    • Accept TOS
    • Say whatever you want to the other things
    • Follow instructions here to put the new keys in the correct places.
